\noindent
The main purpose of this document is to document the simulators requirements and track questions/details
that need to be addressed.

\begin{itemize}

  \item What exactly is the scope of the functionality that needs to be simulated?
  \begin{todolist}
  \item sensors? hardware emulation?
  \item 3d or 2d graph model? what is expected?
  \item does a physics engine need to be implemented/pulled in?
  \end{todolist}

  \item Deployment \& ISA requirements/expectations
  \begin{todolist}
  \item on what hardware does the simulator need to run?
  \item[\done] is cross-compilation to other operating systems required? (linux, windows)
  \item is cross-compilation to different architectures (isa) required? (x86, aarch64, risc-v)
  \item how's the software distributed? (expectations of teachers)
  \item does the code need to be available to the teachers?
  \end{todolist}

\end{itemize}
